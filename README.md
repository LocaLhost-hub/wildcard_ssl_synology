#  Synology Wildcard SSL Automation

Скрипт для автоматического выпуска, установки и продления **Wildcard-сертификатов Let's Encrypt** на устройствах Synology NAS (проверено на DSM 7.3).

Утилита базируется на [acme.sh](https://github.com/acmesh-official/acme.sh) и работает через DNS API вашего регистратора. Она самостоятельно проходит проверку владения доменом, получает ключи и внедряет их в операционную систему DSM, заменяя стандартный сертификат и перезапуская веб-службы.

##  Особенности

* **100% Автоматизация:** Введите данные один раз, всё остальное скрипт сделает сам.
* **Поддержка Wildcard:** Один сертификат защищает основной домен и все возможные поддомены.
* **Нативная интеграция:** Не требует Docker-контейнеров или сложных ручных хуков.
* **Чистота:** Скрипт не оставляет временных файлов и корректно убирает за собой проверочные DNS-записи.

##  Поддерживаемые провайдеры

Скрипт поддерживает работу "из коробки" с популярными DNS-хостингами:

* Timeweb Cloud
* Reg.ru
* Yandex (ПДД)
* Beget
* Selectel
* DNS Exit
* Cloudflare

##  Требования

1. Привязанный к вашему IP-адресу домен.
2. Сгенерированный **API-ключ/токен** в панели управления вашего DNS-провайдера (или логин/пароль для некоторых регистраторов).
3. Включенный доступ по **SSH** на Synology NAS.

---

##  Быстрый запуск

Подключитесь к вашему Synology NAS по протоколу SSH и перейдите в режим суперпользователя:

```bash
sudo -i

```

Выполните команду ниже. Она скачает скрипт из этого репозитория, выдаст нужные права и сразу запустит его:

```bash
curl -s -o /root/wildcard_ssl.sh https://raw.githubusercontent.com/LocalHost-hub/wildcard_ssl_synology/main/wildcard_ssl.sh && chmod +x /root/wildcard_ssl.sh && /root/wildcard_ssl.sh

```

В появившемся интерактивном меню укажите ваш домен, email (для Let's Encrypt) и API-токен (или логин/пароль) выбранного регистратора. Всё остальное скрипт сделает автоматически.

---

##  Настройка автопродления (Cron)

Сертификаты Let's Encrypt действуют 90 дней. Для их автоматического обновления за 30 дней до окончания срока необходимо добавить задачу в системный планировщик DSM.

Выполните эту команду от имени `root`:

```bash
echo -e "0\t3\t*\t*\t*\troot\texport SYNO_Create=1 && export SYNO_USE_TEMP_ADMIN=1 && /root/.acme.sh/acme.sh --cron --home \"/root/.acme.sh\" > /dev/null 2>&1" >> /etc/crontab && synosystemctl restart crond

```

Задача будет запускаться ежедневно в 03:00 ночи. Если сертификат еще свежий — скрипт просто завершит работу. Если требуется продление — он сам обновит ключи через API и перезапустит службы NAS.

---

##  Безопасность и хранение данных

Если вы используете провайдеров, требующих логин и пароль (например, Reg.ru или Beget), либо передаете API-токены, эти данные сохраняются утилитой `acme.sh` локально для последующего автопродления.

Они физически хранятся в конфигурационном файле:
`/root/.acme.sh/account.conf`

* **Это безопасно:** Файл находится в закрытой директории суперпользователя (`root`). Никакие сторонние приложения, Docker-контейнеры или другие пользователи NAS не имеют к нему доступа.
* **Смена пароля:** Если вы измените пароль от личного кабинета у вашего регистратора, автопродление перестанет работать. Вам нужно будет открыть этот файл по SSH (`nano /root/.acme.sh/account.conf`) и вручную обновить сохраненные данные.

---

##  Полное удаление

Если вы хотите полностью удалить `acme.sh`, сертификаты и настройки планировщика из системы, поочередно выполните эти команды от имени `root`:

```bash
# 1. Удаляем задачу из cron и перезапускаем службу
sed -i '/acme.sh/d' /etc/crontab && synosystemctl restart crond

# 2. Удаляем acme.sh из системы
/root/.acme.sh/acme.sh --uninstall

# 3. Удаляем остаточные файлы, ключи и сам скрипт
rm -rf /root/.acme.sh
rm -f /root/wildcard_ssl.sh

```
